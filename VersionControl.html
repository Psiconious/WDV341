Git version control

Version Control

  • Version control is a practice of tracking changes to source code.

Add command

  • Updates index using content found in the working tree. This command can be used multiple times before a commit

Commit command

  • Commits contain the content of the index and a log message describing changes.

Push command

  • Push updates the remote repositories with all the committed changes.

Pull command

  • Pull brings in changes from the remote repository to your local machine.

Clone command

  • Clone copies the remote repository into a new directory
